Chosen from the carry-forward candidate list in the 07-18 SEO pass ("automatic best-discount
selection / preventing code stacking"), sharpened into the highest-pain phrasing of that cluster:
stopping discount codes from applying to sale items. Forum validation is strong and verbatim:
recurring community threads titled "Exclude sale items from discount codes on EXISTING discounts,"
"Discount Stacking Limitations: Inability to Combine Multiple Discount Codes and Automatic
Discounts," and repeated variants of "discount code applying on top of already reduced prices."
Third-party coverage is thin (a couple of app-vendor blogs and the help docs), while demand is
steady, which fits the brief of high demand and low existing content. No existing dispatch covered
it: the closest posts (tiered volume discounts, schedule a sale, bulk price edits, free gift) touch
the discount engine but none addresses code-on-markdown stacking or the missing exclude-sale-items
control.

The native gap is real, specific, and credited. Shopify's combination settings do let product,
order, and shipping discounts be told to stack or not stack, and codes can be scoped by collection,
spend, or segment. But (1) a sale run the way most stores run one, by lowering price and setting a
compare-at price, is not a discount to the discount engine at all, so the combination rules never
see it and a sitewide code lands on markdowns in full; (2) there is no exclude sale items setting
anywhere in the admin; (3) the eligibility workaround (automated collection keyed off compare-at,
codes scoped to the inverse full-price collection) decays with compare-at discipline and cannot
always be applied to codes already issued; (4) popup, loyalty, and review apps mint sitewide codes
outside the merchant's scoping habit; and (5) the old Plus backstop, a Shopify Script that stripped
a code when it met a markdown, stopped executing on June 30, 2026 (timely angle, consistent with the
Scripts-sunset fact used in Nº 84). The playbook frames the differentiator: policy in plain
language, collections kept true on every price change, every new code scoped at creation including
app-minted ones, checkout resolved to the single best discount, the outcome explained to the
shopper, and the judgement calls (service-recovery codes, deliberate campaign overlaps, floor
breaches) escalated to a human.

The pain-point heading screams the use case ("the discount code that double-dips your sale"); the
H1 carries the high-intent search phrasing ("How to stop discount codes from applying to sale items
on Shopify"). Target phrases: "Shopify exclude sale items from discount code," "Shopify discount
code on sale items," "Shopify stop discount stacking," "Shopify double discount," "Shopify discount
combinations," "Shopify prevent discount codes on sale items," "Shopify welcome code sale items,"
and "Shopify discount stacking rules."
