## New post

**Dispatch Nº 64 — "The image alt text no one writes"**
`the-image-alt-text-no-one-writes.html` · by Priya Singh · 2026-06-20 · ~2,000 words

Chosen because image alt text is a high-volume, frequently-complained Shopify pain that the existing
17 dispatches had not covered. The Shopify community forums carry the same recurring question in
near-identical words ("is there a way to bulk upload alt text to images on Shopify"), and the answer
is consistently no native bulk path. It is adjacent to but clearly distinct from the two on-page SEO
posts already published: the meta-description post (Nº 61) is the SERP snippet, the product-descriptions
post (Nº 63) is the on-page body copy, and this post is the alt attribute on the images. The heading
screams the use case for anyone living the pain ("no one writes"), pulling searchers who feel the blank
catalog directly, and it carries the searched phrase "image alt text."

The Shopify-specific limitations are confirmed from current research and form the crux:
(1) There is no bulk editor for alt text in the admin and no audit view, so you set it one image at a
time inside each product's media and you cannot even see which images are missing it. (2) Nothing is
filled in by default; Shopify does not generate a description from the image and the file name is not
alt text, so a photo uploaded as IMG_4821.jpg ships with nothing a screen reader or search engine can
use. (3) The one bulk path is the product CSV's Image Alt Text column, which only moves the data (you
still author every line), covers only product images and not theme/content images, and risks a
re-import overwriting neighboring fields. (4) Cheap bulk apps template the line from product title plus
variant, so the front, the back, and the detail shot all read the same and none describes the actual
photo.

The angle beyond "install a bulk alt text app": the blank field fails two jobs at once. For
accessibility, a screen reader reads the file name aloud (or nothing) instead of the product, and
missing alt text is the single most common accessibility failure cited in the demand letters now
landing on ecommerce stores. For search, the engine cannot read a photo's contents, so a blank alt
keeps product images out of image search and gives the page one less signal; the newer AI answer
engines read the same descriptive text. The differentiator is AI vision: good alt text has to come from
looking at the specific picture, which a product-name template never does. The Dugong playbook finds
every image with missing, duplicated, or file-name alt, looks at the image plus product context, writes
a concise accurate line under ~125 characters in the store's voice, keeps near-identical variant/angle
shots distinct, leaves genuinely decorative images with an empty alt so screen readers skip them, writes
back through the Admin API touching nothing else, re-checks new uploads, and logs each line while
flagging the images too generic to describe well.
