## New post

**Dispatch Nº 54 — "The return that won't approve itself"**
`the-return-that-wont-approve-itself.html` · by Priya Singh · 2026-06-09 · ~1,720 words

Chosen because returns and exchanges are one of the most-complained-about Shopify workflows on
merchant subreddits and the community forums, and the topic was not yet covered by the existing
seven dispatches (address validation, fraud flag, overselling, WISMO, order changes, the field
study, the lead essay). The angle is the specific, real Shopify limitation set: native self-serve
returns still require manual approval of every request (Shopify Flow can't fully automate it),
true exchanges aren't supported (a size/colour swap means refund-and-recreate by hand or a
third-party app), return policies can only key off delivery date and product, and refunds go only
to the original payment method. Because the refund is the path of least resistance and most returns
are sizing/fit issues, sales that could stay sales walk out as money returned. Returns also scale
directly with sales, so the manual queue grows on a good week.

Statistics are sourced from current web research, not invented: ~19–20% ecommerce return rate
(NRF 2025), apparel 20–40%; ~45% of returns are sizing/fit/colour; returns automation sharply
reduces return-related support tickets and speeds processing; an easy return experience makes
shoppers far more likely to buy again. Shopify-specific limitations confirmed across returns-app
and Shopify Help documentation (manual approval, no self-serve exchanges, limited policy rules,
original-payment-method-only refunds).
