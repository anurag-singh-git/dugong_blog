## Resolved: the index `#post` anchor (prior "standing item")

Every prior pass flagged the `DISPATCH OF THE WEEK` featured card on index.html as
`<a href="#post">` with no matching `#post` element, and left it as an inert placeholder. This run
read the navigation script and confirmed it is **not** a dead link. The page is a JS-routed
single-page layout: a global click handler matches any `[data-nav]` element, calls
`preventDefault()`, and swaps views via `showView()` (toggling `#view-home` / `#view-post`). The
featured card carries `data-nav="post"`, so clicking it opens the post view as designed. The
`href="#post"` is only a no-JS fallback target, never followed when scripting is on. Not a defect;
no change needed. The same mechanism drives the brand link, article cards, archive list rows, and
back links, all of which use `data-nav` and work the same way.
