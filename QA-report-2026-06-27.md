# QA report — 2026-06-27 (full-site sweep)

**Scope:** Whole blog — index.html + all 22 dispatch posts + sitemap.xml, feed.xml, llms.txt,
robots.txt, CNAME. Automated scheduled QA run; no user present, executed autonomously.

## Result: one real defect found and fixed (new post under-linked at publish), site otherwise clean

Today's new post, Nº 69 "The tracking number you paste in by hand" (dated JUN 27), shipped at
06:19, three minutes after the 06:16 SEO/GEO audit pass had already declared the site clean against
a 21-post snapshot. As a result the new post missed back-link propagation: at the start of this run
it sat at in-degree 0 (no sibling post linked to it) while every other post was at in-degree 21.
This is the site's recurring defect class — a new post ships before its card is inserted into every
existing post's `.related-grid`. This run completed the propagation and re-verified the whole site.

## Issue found and fixed

**New post Nº 69 was orphaned in the back-link graph.** Inserted a matching `.related-grid` card
pointing to `the-tracking-number-you-paste-by-hand.html` into every sibling post that was missing
it. The card mirrors the established convention exactly: `research` tag, `9 min` read, title "The
tracking number you paste in by hand", the post's canonical meta description as the dek, `JUN 27`
date, `P. Singh` author. Two posts (order-tag, wismo) already carried the card by the time this run
reached them — the publish/SEO pipeline appears to have been finishing propagation concurrently —
so cards were added only where genuinely absent. Post-fix the graph is uniform: in-degree 21 and
out-degree 21 on all 22 posts, every related-grid holds exactly 21 cards, with no duplicate cards
and no self-cards.
